# Remember Me — AI 记忆管家

## 项目需求文档 (PRD)

---

## 1. 项目概述

### 1.1 项目背景

当前 AI 助手（ChatGPT、Claude、DeepSeek 等）存在一个致命缺陷：**没有持久记忆**。每次新对话都是"失忆重启"，用户需要反复解释：
- 自己是谁（角色、经验、领域）
- 做事风格（PRD 结构偏好、语言习惯、详细程度）
- 项目背景（产品名称、目标用户、已做决策）
- 之前的讨论（上周定的方案、修改过的需求）

对于**产品经理、运营、设计师等非技术用户**，这种重复劳动尤其痛苦。他们每周花 6-10 小时写 PRD、方案、文档，AI 能省 4 小时/周，但每次都要重新"培训"AI。

### 1.2 项目定位

**Remember Me** 是一个 VS Code 插件，让非技术用户与 AI 协作时，AI 能"记住"你的做事风格、偏好和过往对话，不再每次从头教。

### 1.3 目标用户

| 用户类型 | 场景 | 痛点 |
|----------|------|------|
| 产品经理 | 写 PRD、需求文档、竞品分析 | 每次都要重新说明产品背景、PRD 模板 |
| 运营人员 | 写活动方案、数据分析报告 | 需要反复解释品牌调性、目标人群 |
| 设计师 | 写设计说明、用户调研报告 | 每次都要说明设计规范、审美偏好 |
| 学生/研究者 | 写论文、文献综述 | 需要反复说明研究领域、方法论偏好 |
| 创业者 | 写商业计划书、融资材料 | 每次都要重新介绍商业模式、市场定位 |
| 非技术管理者 | 写汇报材料、战略文档 | 需要反复说明管理风格、汇报格式 |

### 1.4 核心价值主张

> **"一次设定，永久记住。让 AI 真正成为你的协作伙伴，而不是每次都要重新认识的陌生人。"**

## 2. 功能需求

### 2.1 记忆分层体系

Remember Me 采用四层记忆结构，覆盖用户与 AI 协作的所有维度：

#### 2.1.1 个人画像（Profile）

**定义：** 你是谁？你的背景、经验、领域。

**包含内容：**
- 身份角色：产品经理 / 运营 / 设计师 / 学生 / 创业者 / 管理者
- 经验水平：新手 / 1-3年 / 3-5年 / 5年以上
- 行业领域：电商 / SaaS / 社交 / 金融 / 教育 / 医疗 / 其他
- 专业背景：技术 / 商业 / 设计 / 文科 / 其他

**使用场景：**
- 写 PRD 时，AI 知道你是 B 端 SaaS 产品经理，不会给你 C 端消费品的建议
- 写论文时，AI 知道你是计算机专业研究生，不会用太基础的术语

#### 2.1.2 做事风格（Style）

**定义：** 你怎么做？你的偏好、习惯、标准。

**包含内容：**
- 文档结构偏好：先写背景还是先写功能点？
- 详细程度：简洁（1页）/ 标准（3-5页）/ 详尽（10页以上）
- 语言风格：正式 / 口语化 / 学术
- 语言：中文 / 英文 / 双语
- 特殊习惯：
  - PRD：是否必须包含用户故事、竞品对比、验收标准？
  - 论文：引用格式（APA / MLA / GB/T 7714）？
  - 商业计划书：是否必须包含财务预测、市场分析、竞争壁垒？
  - 汇报材料：是否必须包含数据图表、SWOT 分析？
- 回复风格：先给框架再填细节 / 直接给完整内容 / 逐步引导

**使用场景：**
- 你习惯用 MoSCoW 法标注优先级，AI 自动在 PRD 中标注
- 你写论文必须用 GB/T 7714 引用格式，AI 自动规范引用
- 你写商业计划书必须包含 3 年财务预测，AI 自动补充

#### 2.1.3 项目上下文（Project Context）

**定义：** 你在做什么？当前项目的背景、决策、进展。

**包含内容：**
- 项目名称、目标用户、核心功能
- 已确定的架构决策（如"认证用 OAuth 2.0"）
- 已讨论过的方案（如"上周决定用微服务架构"）
- 已定义的术语表（如"用户 = 企业管理员，不是终端用户"）
- 竞品信息（如"主要竞品是 Slack 和飞书"）

**使用场景：**
- 你在写"用户管理"模块的 PRD，AI 自动知道你的"用户"指的是企业管理员
- 你在写论文第三章，AI 自动引用第一章定义的理论框架

#### 2.1.4 对话历史（Conversation History）

**定义：** 你们聊过什么？之前的讨论、决策、修改记录。

**包含内容：**
- 对话主题、日期
- 关键决策（如"决定支持手机号+验证码登录"）
- 新发现（如"用户更关注安全合规而非用户体验"）
- 修改记录（如"根据反馈，将优先级从 Must 改为 Should"）
- 提取的洞察（AI 自动总结）

**使用场景：**
- 你上周和 AI 讨论过"用户角色体系"，这周写"权限管理"时 AI 自动提醒
- 你之前修改过某个方案，AI 记得修改原因，避免重复讨论

---

### 2.2 记忆触发机制（混合模式）

#### 2.2.1 自动基础记忆（🟢 每次对话自动注入）

**触发条件：** 每次启动 AI 对话时

**注入内容：**
- 个人画像（你是谁）
- 当前项目上下文（你在做什么）
- 做事风格中的"硬性规则"（如必须用 MoSCoW）

**用户感知：** 对话顶部显示"🧠 Remember Me 已激活：B端SaaS产品经理 | 项目：TeamFlow"

#### 2.2.2 智能推荐记忆（🟡 根据内容自动推荐）

**触发条件：** AI 检测到当前内容与历史记忆相关时

**推荐内容：**
- 相关历史对话（"你上周讨论过用户角色体系，要查看吗？"）
- 相关决策（"你之前决定用 OAuth 2.0，这个方案需要调整吗？"）
- 相关术语（"你定义的'用户'是指企业管理员，需要确认吗？"）

**用户感知：** 对话内显示"💡 相关记忆：你上周讨论过... [查看] [忽略]"

#### 2.2.3 手动检索记忆（🔴 用户主动搜索）

**触发条件：** 用户主动搜索

**搜索方式：**
- 关键词搜索（"用户权限"）
- 时间范围搜索（"上周的讨论"）
- 项目筛选（"TeamFlow 项目"）
- 标签筛选（"已决策 / 待确认 / 已废弃"）

**用户感知：** 侧边栏搜索框，输入后显示相关记忆片段

### 2.4 支持的多场景写作

Remember Me 不仅支持 PRD，还支持以下场景：

| 场景 | 记忆重点 | 示例 |
|------|----------|------|
| **PRD（产品需求文档）** | 产品背景、用户画像、功能结构、验收标准 | 写登录功能 PRD 时，AI 知道你的用户是企业管理员 |
| **商业计划书** | 商业模式、市场定位、财务模型、竞争壁垒 | 写融资材料时，AI 记得你的目标市场是东南亚中小企业 |
| **学术论文** | 研究领域、理论框架、方法论、引用格式 | 写第三章时，AI 自动引用第一章的文献综述 |
| **市场调研报告** | 调研方法、样本特征、分析框架、结论 | 写竞品分析时，AI 记得你定义的评估维度 |
| **活动策划方案** | 品牌调性、目标人群、预算范围、渠道偏好 | 写双11方案时，AI 知道你的品牌是高端路线 |
| **设计说明文档** | 设计规范、色彩体系、组件库、交互原则 | 写组件说明时，AI 知道你的设计系统是 Ant Design |
| **技术方案文档** | 技术栈、架构决策、接口规范、部署环境 | 写 API 设计时，AI 记得你的认证方式是 JWT |
| **汇报材料** | 汇报对象、关注重点、数据口径、格式要求 | 写周报时，AI 知道你的老板只关心营收数据 |

### 3.2 存储方案

**采用纯 JSON 文件存储，原因：**
- 非技术用户可以看到、理解、手动修改自己的记忆
- 零依赖，不需要安装数据库
- 可以用 Git 备份
- 符合"透明档案"的理念

**文件结构：**
```
~/.remember-me/
├── profile.json                   # 个人画像 + 做事风格（全局）
├── projects/
│   ├── teamflow/                  # 每个项目一个文件夹
│   │   ├── context.json           # 项目上下文
│   │   └── conversations/
│   │       ├── 2026-07-08-prd-login.json
│   │       └── 2026-07-09-user-roles.json
│   └── thesis-llm/                # 论文项目
│       ├── context.json
│       └── conversations/
└── templates/
    ├── prd-standard.json          # PRD 模板
    ├── business-plan.json         # 商业计划书模板
    └── thesis-template.json       # 论文模板
```

## 4. 用户旅程

### 4.1 首次使用（Onboarding）

**Step 1：安装插件**
- 在 VS Code 扩展市场搜索 "Remember Me"
- 点击安装

**Step 2：启动设置向导**
- 安装后自动弹出欢迎页面
- 3 分钟问卷式设置：
  - "你是做什么的？"（产品经理 / 运营 / 设计师 / 学生 / 创业者 / 其他）
  - "你主要写什么类型的文档？"（PRD / 商业计划书 / 论文 / 其他）
  - "你的文档风格是？"（简洁 / 标准 / 详尽）
  - "你有什么特殊习惯？"（多选：MoSCoW优先级 / 用户旅程图 / 竞品对比 / 财务预测 / 引用格式规范）

**Step 3：创建第一个项目**
- "你目前在做什么项目？"
- 输入项目名称、目标用户、核心功能

**Step 4：开始使用**
- 打开任意文件，点击状态栏 "🧠 Remember Me" 图标
- 开始 AI 对话，体验"AI 记得你"的惊喜

### 4.2 日常使用

**场景 1：写 PRD**
1. 打开 `login-prd.md`
2. 点击 "🧠 开始对话"
3. AI 自动注入："你是 B 端 SaaS 产品经理，项目 TeamFlow，风格是详细 PRD + MoSCoW"
4. 你说："帮我写登录功能的 PRD"
5. AI 回复：自动包含背景、用户故事、功能描述、验收标准、竞品对比，并标注 MoSCoW 优先级
6. AI 提示："💡 你之前决定用 OAuth 2.0 + SSO，这个方案需要调整吗？"

**场景 2：写商业计划书**
1. 打开 `business-plan.md`
2. 切换项目为 "MyStartup"
3. AI 自动注入："你是创业者，项目 MyStartup，目标市场东南亚中小企业"
4. 你说："帮我写市场分析章节"
5. AI 回复：自动包含东南亚中小企业市场规模、增长趋势、竞争格局
6. AI 提示："💡 你之前定义的竞争优势是'本地化客服'，要在这里强调吗？"

**场景 3：写论文**
1. 打开 `chapter-3.md`
2. 切换项目为 "Thesis-LLM"
3. AI 自动注入："你是计算机专业研究生，研究领域是大语言模型，理论框架是 Transformer"
4. 你说："帮我写实验设计部分"
5. AI 回复：自动引用第一章的文献综述，使用 GB/T 7714 引用格式
6. AI 提示："⚠️ 你之前定义的评价指标是 BLEU 和 ROUGE，要补充新的指标吗？"

### 4.3 记忆更新

**自动更新：**
- AI 检测到新信息时，提示用户确认
- 用户点击"更新"后，自动写入 JSON 文件

**手动更新：**
- 点击状态栏 "🧠" → "编辑记忆"
- 打开可视化记忆面板，直接修改

**版本控制：**
- 每次更新自动保存历史版本
- 可回滚到任意版本

## 5. 商业化策略

### 5.1 开源版（免费）

- 本地 JSON 存储
- 基础记忆注入（Profile + Project Context）
- 对话历史记录（最近 10 条）
- 支持 1 个 AI 提供商
- 单设备使用

### 5.2 Pro 版（付费）

- **云端同步**：多设备记忆同步，加密存储
- **无限历史**：无限对话历史记录
- **智能推荐**：AI 自动推荐相关记忆
- **语义搜索**：基于向量数据库的高级搜索
- **团队协作**：共享项目记忆，权限管理
- **记忆优化**：AI 自动分析记忆质量，提出优化建议
- **模板市场**：社区共享的 PRD / 商业计划书 / 论文模板
- **优先支持**：专属客服，快速响应

### 5.3 企业版（定制）

- 私有化部署
- 企业 SSO 集成
- 审计日志
- 自定义 AI 模型接入
- API 接口

## 6. 竞品分析

| 竞品 | 类型 | 记忆能力 | 定位 | 缺点 |
|------|------|----------|------|------|
| **ChatGPT** | 云端对话 | 无持久记忆 | 通用 AI | 每次新对话失忆 |
| **Claude** | 云端对话 | 无持久记忆 | 通用 AI | 扩展升级丢失历史 |
| **Claude Code** | IDE 插件 | 无持久记忆 | 编程助手 | 对话仅存内存，升级丢失 |
| **Mem0** | 记忆框架 | 有记忆 | 开发者工具 | 需要 Docker/PostgreSQL，太重 |
| **Letta** | 记忆框架 | 有记忆 | 开发者工具 | 需要复杂基础设施 |
| **Continue** | VS Code 插件 | 无持久记忆 | 编程助手 | 仅代码补全，无风格记忆 |
| **ChatPRD** | Web 工具 | 无持久记忆 | PRD 生成 | 仅 PRD，无跨场景记忆 |
| **Productboard Spark** | 企业工具 | 有产品上下文 | 企业级 | 太重，不适合个人 |

**Remember Me 的差异化：**
- 唯一面向**非技术用户**的 AI 记忆工具
- 唯一支持**多场景写作**（PRD / 商业计划书 / 论文 / 方案）
- 唯一采用**透明 JSON 存储**，用户完全掌控自己的记忆
- 唯一**开源核心 + 闭源商业化**的可持续模式

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AI 提供商 API 变更 | 高 | 统一接口设计，快速适配 |
| 用户隐私担忧 | 中 | 本地存储优先，透明 JSON，用户完全掌控 |
| 竞品快速跟进 | 中 | 先发优势，社区生态，持续迭代 |
| 技术门槛过高 | 低 | 非技术用户优先设计，3 分钟上手 |
| 记忆质量不稳定 | 中 | 用户确认机制，版本控制，人工审核 |

### 10.1 记忆注入 Prompt 模板

```markdown
你是用户的 AI 协作助手。以下是关于这位用户的背景信息，请在回复中严格遵循这些偏好：

【身份】
- 角色：{role}
- 经验：{experience}
- 领域：{industry}

【做事风格】
- 文档结构：{structure}
- 详细程度：{detail_level}
- 语言：{language}
- 语气：{tone}
- 特殊习惯：{special_habits}
- 回复风格：{response_style}

【当前项目】{project_name}
- 目标用户：{target_users}
- 核心功能：{core_features}
- 已确定决策：
{decisions}

【相关历史】
{relevant_history}

请基于以上信息协助用户，确保回复符合用户的风格和项目上下文。
```
