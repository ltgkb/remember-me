## 三、核心实现详解

### 3.1 bge-m3 模型选型对比（B1）

```
中文查询 ──► 编码模型 ──► 向量索引 ──► Top-5 召回
              │                │
              ▼                ▼
        all-MiniLM-L6-v2   100 条混合记忆
        bge-m3              (20 英文 + 80 中文)
        bge-small-zh
```

**关键结论**：
- `all-MiniLM-L6-v2`：中→英召回 0.2/5，英→中 2.0/5，跨语言能力严重不足
- `bge-m3`：中→英 3.4/5，英→中 4.2/5，跨语言召回显著改善，满足生产可用标准
- `bge-small-zh`：中文同语言召回 0.785 优于 MiniLM，但跨语言仅 1.0/5，定位纯中文轻量场景

**决策记录**：
- **短期（v0.3.x）**：保持 all-MiniLM-L6-v2，在 UI 标注「Beta」管理预期
- **中期（v0.4.0）**：迁移到 bge-m3，模型体积 2.2GB 在 Pro 版目标设备上可接受

### 3.2 云端同步架构设计（D1）

基于 PRD §4.2 Pro 版需求，完成端到端加密同步架构设计：

- **加密**：AES-256-GCM，单文件级，每文件独立 IV；主密钥由 PBKDF2/Argon2 本地派生，服务端零知识
- **同步**：Lamport 时间戳冲突检测，Last-Write-Wins 默认 + 用户手动合并关键文件；增量同步基于内容哈希分块
- **存储**：四方案对比（MinIO / S3+KMS / OSS+KMS / R2），推荐 Cloudflare R2 作为初创期 Pro 版后端
- **合规**：覆盖 GDPR / 个人信息保护法 / Right to Erasure
