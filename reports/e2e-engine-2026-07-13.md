## 1. 检查清单

### 1.1 Python 代码审查

#### `packages/memory-engine/src/memory_engine/server.py` (401 行)
- **实现方式**: 基于 Python 标准库 `http.server.BaseHTTPRequestHandler` 的单线程 HTTP 服务
- **监听地址**: 默认 `127.0.0.1:8765`，支持 `--host` 和 `--port` 参数自定义
- **CORS 支持**: 已配置 `Access-Control-Allow-Origin: *`，支持 `GET, POST, OPTIONS`
- **端点清单**:
  | 方法 | 路径 | 功能 |
  |------|------|------|
  | GET | `/health` | 健康检查，返回 `{"status":"ok","service":"remember-me-engine","version":"0.1.0"}` |
  | POST | `/extract` | 接收文本，调用 `InfoExtractor.extract()` 提取关键信息 |
  | POST | `/search` | 基于关键词在数据目录的 `.json`/`.json.gz` 文件中做字符串匹配 |
  | GET | `/backups` | 列出指定文件的备份历史 |

- **异常处理**: 请求体 JSON 解析失败返回空字典；缺少必填字段返回 400；数据目录不存在返回 404
- **日志输出**: 重写 `log_message` 到 stderr，保留时间戳

#### `packages/memory-engine/src/memory_engine/extractor.py` (389 行)
- **核心类**: `InfoExtractor` — 纯函数风格，不维护内部状态
- **提取类型**: `decision`(决策)、`term`(术语)、`todo`(待办)、`date`(日期)、`role`(角色)
- **置信度算法**: 基础分 0.5 + 句首位置加分(0.15) + 长度评分(高斯型) + 关键词密度加权(0.10)
- **去重策略**: 相同 `raw_text` 保留置信度最高的一条
- **洞察生成**: `generate_insights()` 聚合提取结果，输出分类汇总和潜在风险提示（密集决策、术语定义、待办任务、多方参与、时间敏感）

| 服务端字段 | 客户端字段 | 映射代码位置 | 说明 |

## 4. 发现的问题与建议

### 4.1 已知问题

| 优先级 | 问题 | 影响 | 建议 |
|--------|------|------|------|
| Low | `/search` 目前仅为**字符串子串匹配**，非语义搜索 | 无法召回同义词/近义词结果 | Phase 4 引入向量索引后替换为语义搜索 (已在原型中验证可行性) |
| Low | `extractor.py` 的 `_ROLE_RE` 会将 "Python" 识别为角色而非技术栈 | 部分提取类型不够精确 | 可考虑增加 `tech_stack` 类型或优化角色正则 |
| Medium | 服务端使用单线程 `http.server`，高并发时可能成为瓶颈 | 仅影响本地开发环境 | 生产环境建议迁移至 `uvicorn` + `FastAPI` (已在 PRD 中规划) |
| Low | 数据目录默认在 `~/.remember-me`，Windows 路径含空格时无引号处理 | 潜在的 CLI 调用问题 | 已在 `cli.py` 中使用 `pathlib`，风险可控 |
