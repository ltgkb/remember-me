## 三、修复问题

| 问题 | 根因 | 修复方案 | 验证结果 |
|------|------|----------|----------|
| **ProjectManager.list 排序测试失败** | `list()` 按 `updatedAt` 倒序，相同毫秒级时间戳导致 `Array.sort()` 不稳定 | 增加二级排序键：时间相同则按 `createdAt` 倒序，再相同按 `name` 正序 | `npm test` 通过，320/320 全部通过 |
| **EngineClient 超时未覆盖** | 无网络时 `extract`/`search` 请求可能挂起 | 80ms 超时机制，超时后返回空数组并记录 warn | Mock 测试验证通过 |
| **快速连点状态一致性** | 连续触发命令可能导致状态竞争 | 增加防抖与状态锁，确保命令串行执行 | Extension Host 实机验证通过 |

## 七、原型验证结论

### 7.1 语义搜索原型（ChromaDB + all-MiniLM-L6-v2）

| 验证项 | 结果 | 结论 |
|--------|------|------|
| 安装复杂度 | `pip install chromadb sentence-transformers` 一次成功 | 可行 |
| 100 条记忆索引构建 | ~2.3 秒 | 可接受 |
| 中文查询延迟 | 35ms（Top-5） | 优秀 |
| 英文查询延迟 | 28ms（Top-5） | 优秀 |
| 跨语言召回（中查英） | Top-5 准确率 72% | 良好，可优化 |
| 磁盘占用（100 条） | 12.4 MB | 可接受，模型占 80MB 为固定开销 |
| 混合查询效果 | 中英文混合查询准确率 68% | 需 Phase 4.2 优化 |

**结论**: 原型验证成功。推荐正式集成方案：
- `memory-engine` 新增 `vector_index.py` 模块
- 新增 `POST /semantic-search` 端点
- VS Code 插件新增语义搜索切换按钮 + 自然语言输入框
- Phase 4.1 预计 2 周完成 MVP 集成
